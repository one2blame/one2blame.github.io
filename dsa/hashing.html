Hashing
=======

A hash map is an unordered data structure that stores key-value pairs. A hash map can add and remove
elements in O(1), constant, time as well as update values associated with a key and check if a key
exists, also in O(1) time. You can iterate over both the keys and values of a hash map, but the
iteration won't necessarily follow any order (there are many implementations and this is language
dependent for the built-in types).

The following operations are all O(1) for a hash map:

  * Add an element and associate it with a value
  * Delete an element if it exists
  * Check if an element exists

Hash maps maintain the same time complexity as arrays for the following operations:

  * Find length/number of elements
  * Updating values
  * Iterate over elements


_ Disadvantages:

The biggest disadvantage of hash maps is that for smaller input sizes, they can be slower due to
overhead. Every key must be processed by the hash function prior to creating a key for storage. Hash
maps also take up more space because hash tables are implemented using a fixed size array. If a hash
table must be resized, all existing keys must be re-hashed, and the hash table may use an array
significantly larger than the number of elements it contains.

Collisions are another disadvantage of hash maps. A collision occurs when two keys are hashed to the
same index in the array. There are many ways to handle collisions, but the most common is to use
linked lists to store all keys that hash to the same index.

_ Sets:

A set is a hash map that only stores keys, not values. Sets are useful for storing unique elements
and checking if an element exists. Sets are also useful for removing duplicates from an array.


Checking for Existence
======================

The most common use case for a hash map is to check if an element exists in O(1) time. Anytime you
find your algorithm running "if ..." "in ...", then consider using a hash map or set to store
elements to have these operations run in O(1) time.
